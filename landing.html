Features

Everything you need to ship infrastructure.

Visual-first workflow. Schema-driven forms. Real-time code generation. All in the browser.

✎

Drag & Drop Canvas

Infinite, zoomable canvas. Drag cloud services from the library, connect them, group them into logical units that map to Terragrunt folders.

</>

Real-time HCL Generation

Every change on the canvas triggers instant code regeneration. Preview main.tf, variables.tf, outputs.tf, and terragrunt.hcl files live.

⚙

Schema-Driven Forms

Dynamic property panels with typed inputs, enum dropdowns for instance types and runtimes, and secrets management with reveal toggles.

✓

Built-in Validation

Catch issues before deploy — circular dependencies, missing required inputs, invalid output mappings. Visual red/yellow rings on problematic nodes.

⇄

Dependency Engine

Edges auto-generate dependency blocks with proper config paths. Wire source outputs to target inputs through a visual mapping UI.

☰

Export Everything

Download a production-ready .zip with Terragrunt structure, CI/CD pipelines (GitHub Actions or GitLab), or export as PNG, JPG, JSON.

How it works

Three steps to production IaC.

1

Design

Drag AWS, Azure, or GCP services onto the canvas. Group them into logical modules. Connect them with edges.

2

Configure

Set instance types, runtimes, secrets, and tags through dynamic forms. The code preview updates in real time.

3

Deploy

Export a git-ready Terragrunt project with CI/CD pipeline. Push to your repo. terragrunt run-all apply.
